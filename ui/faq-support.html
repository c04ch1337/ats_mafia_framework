Frequently Asked Questions

Getting Started

What are the system requirements for ATS MAFIA?

ATS MAFIA requires:

  • Hardware: 8GB RAM minimum (16GB recommended), 2 CPU cores minimum
  • Software: Docker and Docker Compose, modern web browser
  • API Keys: At least one LLM provider API key (OpenAI, Anthropic, or Google)
  • Storage: 10GB free disk space

How do I get started with my first training session?

Follow these steps:

  1. Complete the 5-minute Quick Start Guide
  2. Create your first profile (Red Team Operator recommended)
  3. Choose a beginner scenario like "Basic Penetration Test"
  4. Set a small budget ($5-10) for your first session
  5. Launch the session and monitor progress in the Situation Room

Which LLM provider should I use?

Each provider has different strengths:

  • OpenAI GPT-4: Best overall performance, higher cost
  • Anthropic Claude: Excellent for reasoning, moderate cost
  • Google Gemini: Good value, cost-effective option

We recommend starting with Google Gemini for cost efficiency, then upgrading to GPT-4 for advanced scenarios.

Cost Management

How much does a typical training session cost?

Costs vary by complexity:

  • Beginner scenarios: $5-15 per session
  • Intermediate scenarios: $15-30 per session
  • Advanced scenarios: $30-60 per session
  • Expert scenarios: $60-100+ per session

Always set budget limits to control costs!

How can I reduce training costs?

Several strategies to optimize costs:

  • Use cost-effective models for routine tasks
  • Set appropriate session budgets
  • Monitor costs in real-time during sessions
  • Use the cost optimization recommendations
  • Start with simpler scenarios and progress gradually

What happens if I exceed my budget?

The system provides multiple safeguards:

  • Alerts at 75% and 90% of budget
  • Automatic session pause at 100% budget
  • Option to continue with additional budget approval
  • Detailed cost breakdown for analysis

Technical Issues

Why is my WebSocket connection failing?

Common WebSocket issues and solutions:

  • Port blocked: Ensure port 8080 is accessible
  • Firewall: Check firewall settings
  • Container not running: Verify with docker-compose ps
  • Network issues: Try refreshing the page

Why are containers restarting repeatedly?

Container restart loops usually indicate:

  • Configuration errors: Check .env file settings
  • Missing dependencies: Verify Docker image build
  • Port conflicts: Check for port conflicts
  • Resource limits: Monitor system resources

Use docker logs [container-name] to diagnose specific issues.

How do I reset my system configuration?

To reset configuration:

  1. Stop all containers: docker-compose down
  2. Remove volumes (WARNING: deletes all data): docker-compose down -v
  3. Recreate .env from template: cp .env.example .env
  4. Restart: docker-compose up -d

Training & Performance

How do I improve my training scores?

Tips for better performance:

  • Start with appropriate difficulty levels
  • Review after-action reports carefully
  • Focus on learning objectives, not just completion
  • Practice consistently to build skills
  • Use the analytics dashboard to identify weak areas

What do the performance metrics mean?

Key metrics explained:

  • Success Rate: Percentage of objectives completed
  • Efficiency: Cost-to-results ratio
  • Learning Velocity: Rate of skill improvement
  • Adaptability: Performance in novel situations
  • Consistency: Performance variance across sessions

How do I create custom scenarios?

Creating custom scenarios:

  1. Define clear objectives and success criteria
  2. Set appropriate difficulty level
  3. Configure environment and constraints
  4. Test the scenario yourself
  5. Publish for team use

See the User Guide for detailed scenario creation instructions.

Common Troubleshooting

Docker Issues

  1. Check Docker is running: docker --version
  2. Verify Docker Compose: docker-compose --version
  3. Check container status: docker-compose ps
  4. View logs: docker-compose logs [service]
  5. Restart services: docker-compose restart

Network Issues

  1. Check port availability: netstat -an | grep :8000
  2. Verify firewall settings
  3. Test API connectivity: curl http://localhost:8000/health
  4. Check WebSocket: wscat -c ws://localhost:8080
  5. Verify DNS resolution

Database Issues

  1. Check PostgreSQL status: docker-compose exec postgres pg_isready
  2. Verify connection string in .env
  3. Check database logs: docker-compose logs postgres
  4. Test connection: docker-compose exec postgres psql -U ats_user -d ats_mafia
  5. Restart database: docker-compose restart postgres

LLM API Issues

  1. Verify API key in .env file
  2. Test API key with curl
  3. Check API service status
  4. Verify rate limits and quotas
  5. Check billing status

Performance Issues

  1. Check system resources: docker stats
  2. Monitor memory usage
  3. Check CPU utilization
  4. Review disk space: df -h
  5. Optimize container resource limits

Authentication Issues

  1. Clear browser cache and cookies
  2. Check token expiration
  3. Reset user credentials
  4. Restart authentication service
